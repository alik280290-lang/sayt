01 — Подготовка

Что нужно собрать до начала

Claude не придумает за тебя контент. Всё что он делает — превращает твой материал в работающий код. Поэтому сначала собери:

  • Видео работ — загрузи на Kinescope (бесплатно, до 100 ГБ). Форматы MP4, до 10 МБ каждое.
  • Список клиентов и проектов — названия, что делал, результаты. Даже тезисно.
  • Награды и достижения — если есть. У меня: 1-е место AI Realism + People's Choice, VIDMK 2026.
  • Прайс — три тарифа или диапазон цен.
  • Контакты — Telegram, email.
  • 10–15 сайтов-референсов — что нравится визуально. Pinterest, Awwwards, Behance.

Kinescope даёт embed-ссылку в формате kinescope.io/embed/[ID]. Именно её вставляем в iframe на сайте.

02 — Инструменты

Стек этого сайта

Никаких React, никаких сборщиков. Один файл — один сайт.

Claude Code
AI пишет код по твоим промптам. Нужна подписка Pro или Max.
VS Code
Редактор кода. Claude Code работает прямо внутри него.
GSAP + ScrollTrigger
Плавные анимации при скролле. Подключается через CDN, ничего не устанавливать.
Lenis
Инерционный скролл — страница «плывёт» а не прыгает.
Kinescope
Хостинг видео с русским интерфейсом. Даёт iframe-embed без рекламы.
GitHub Pages
Бесплатный хостинг. Файл в репозитории = живой сайт.

04 — Секции по одной

Итеративная доработка

После первой версии дорабатывай каждую секцию отдельным промптом. Не проси переписать весь сайт — только точечные изменения.

Горизонтальный скролл работ (pin-эффект):

ПромптСекция #works: сделай горизонтальный пин-скролл. При скролле вниз камера едет вправо, показывая карточки работ одну за другой. Каждая карточка — на весь экран (100vw × 100vh). Фон — видео через iframe (Kinescope embed). Текст поверх видео внизу слева: название проекта, мета-данные, цитата клиента. Используй GSAP ScrollTrigger с pin: true.

05 — Видео на сайте

Kinescope вместо YouTube

YouTube показывает рекламу и «похожие видео» — это убивает премиальный вид. Kinescope даёт чистый плеер без лишнего.

1
Загрузи видео на kinescope.io

Бесплатный аккаунт, русский интерфейс, 100 ГБ хранилища.

2
Скопируй ID из ссылки

После загрузки ссылка выглядит как kinescope.io/4si6LKEhDC3... — вот это и есть ID.

3
Вставь embed в iframe

Формат: https://kinescope.io/embed/[ID]?autoplay=1&muted=1&loop=1

4
Используй data-src для ленивой загрузки

Вместо src пиши data-src — видео загрузится только когда карточка появится на экране. Это ускоряет сайт.

Видео должно быть сжато: H.264, до 10 МБ, с флагом +faststart. Иначе будет долго грузиться на мобильном.

06 — Деплой

GitHub Pages — бесплатный хостинг

Сайт живёт на GitHub. Каждое изменение — это git push, и через минуту сайт обновлён.

1
Создай репозиторий на github.com

Имя: sayt (или любое). Публичный. Без README.

2
Включи GitHub Pages

Settings → Pages → Source: Deploy from branch → main → / (root).

3
Пушь изменения через Claude Code

Скажи Claude: «запушь изменения на GitHub». Он сделает git add, commit, push за тебя.

После push жди 1–3 минуты. Если сайт не обновился — обнови страницу через Cmd+Shift+R (жёсткое обновление кэша).

07 — Частые ошибки

На чём спотыкаются

  • Видео не играет автоматически — браузер требует muted=1 для autoplay. Без него — тишина и ничего.
  • Анимации не работают — GSAP и Lenis нужно инициализировать после загрузки DOM. Весь JS — перед закрывающим </body>.
  • Сайт не обновился после push — жёсткое обновление Cmd+Shift+R или подождать 2–5 минут.
  • Шрифты не грузятся — Google Fonts нужно подключить в <head>, иначе анимации текста сломаются.
  • Мобильный скролл — горизонтальный pin-эффект на мобильном нужно отключать. Добавь медиа-запрос.
  • Долгая загрузка — используй data-src вместо src для iframe с видео. Иначе все видео грузятся сразу.
08 — Принципы работы с Claude

Как формулировать задачи

  • Один запрос — одно изменение. «Поменяй цвет кнопки на медный» работает лучше чем «переделай весь header».
  • Показывай что не нравится конкретно. «Заголовок слишком маленький на мобильном» → Claude исправит. «Сделай лучше» → непонятно.
  • Сохраняй удачные промпты. Если что-то сработало идеально — запиши, используй снова.
  • Делай коммит после каждого удачного изменения. Так можно откатиться если что-то сломается.
  • Не бойся спрашивать «почему». Claude объяснит любой кусок кода простыми словами.

Главный навык — не знание кода, а умение точно описать что хочешь. Как техзадание дизайнеру, только для AI.

09 — Чеклист перед запуском

Перед тем как отправить ссылку

  • Видео — сжаты, грузятся быстро, используется data-src
  • Мобильная версия — проверена на iPhone в Safari
  • Title и description — заполнены в <head>
  • og:image — есть превью для соцсетей (1200×630px)
  • Контакты — все ссылки рабочие (Telegram, email)
  • Форма — отправка работает (Web3Forms или аналог)
  • Анимации — не мешают читать контент, не тормозят
